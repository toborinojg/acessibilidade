Elementos obsoletos

Especificações baseadas nos documentos do W3C.

Documento raiz

O elemento raiz de todo documento HTML.

<html> Raiz do documento HTML

O elemento <html> representa a raiz de um documento HTML. Todos os outros elementos devem ser descendentes deste elemento. O atributo lang define o idioma do documento.

<!DOCTYPE html>
<html lang="pt-BR">
  <head>...</head>
  <body>...</body>
</html>

Metadados do documento

Contêm informações sobre o documento: título, scripts, folhas de estilo e outros metadados.

<head> Contêiner de metadados

O elemento <head> contém informações legíveis por máquina sobre o documento, como título, scripts e folhas de estilo. Não é renderizado visualmente.

<head>
  <meta charset="UTF-8">
  <title>Minha Página</title>
</head>
<title> Título do documento

Define o título do documento, exibido na aba do navegador e nos resultados de busca. Deve ser único e descritivo.

<title>Elementos HTML 5.0 — Referência Completa</title>
<base> URL base do documento

Especifica a URL base para URLs relativas no documento e o contexto padrão de navegação para links. Deve ser único no <head>.

<base href="https://exemplo.com/" target="_blank">
<link> Recurso externo

Especifica relações entre o documento e recursos externos. Mais usado para vincular folhas de estilo CSS, ícones e fontes.

<link rel="stylesheet" href="estilos.css">
<link rel="icon" href="favicon.ico" type="image/x-icon">
<link rel="preconnect" href="https://fonts.googleapis.com">
<meta> Metadados

Representa metadados que não podem ser representados por outros elementos HTML relacionados a metadados. Usado para charset, viewport, descrição, Open Graph etc.

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="Referência de elementos HTML 5.0">
<meta property="og:title" content="Elementos HTML 5.0">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<style> Estilos CSS inline

Contém informações de estilo CSS aplicadas ao documento. Pode ser colocado no <head> ou no corpo do documento.

<style>
  body {
    font-family: 'Roboto', sans-serif;
    background: #bbddff;
  }
  h1 { color: #003366; }
</style>

Separação de conteúdo

Organizam o conteúdo do documento em seções lógicas, permitindo criar um esquema de estrutura para o conteúdo.

<body> Corpo do documento

Representa o conteúdo principal de um documento HTML. Só pode haver um elemento <body> por documento.

<body onload="iniciar()">
  <h1>Bem-vindo</h1>
  <p>Conteúdo principal aqui.</p>
</body>
<article> Conteúdo independente

Representa uma composição autossuficiente, que poderia ser distribuída e reutilizada de forma independente, como um post de blog, artigo de revista ou comentário.

<article>
  <h2>Título do Artigo</h2>
  <p>Publicado em <time datetime="2025-04-20">20 de abril de 2025</time></p>
  <p>Conteúdo do artigo...</p>
</article>
<section> Seção genérica

Representa uma seção genérica de um documento. Use quando não houver elemento semântico mais específico. Geralmente inclui um cabeçalho.

<section>
  <h2>Introdução</h2>
  <p>Esta seção apresenta o tema...</p>
</section>
<nav> Navegação

Representa uma seção com links de navegação — menu principal, índice, breadcrumb etc.

<nav aria-label="Menu principal">
  <ul>
    <li><a href="/">Início</a></li>
    <li><a href="/sobre">Sobre</a></li>
    <li><a href="/contato">Contato</a></li>
  </ul>
</nav>
<aside> Conteúdo tangencial

Representa conteúdo indiretamente relacionado ao conteúdo principal: barras laterais, callouts, publicidade ou grupos de elementos de navegação.

<aside>
  <h3>Leia também</h3>
  <ul>
    <li><a href="/css">Referência CSS</a></li>
  </ul>
</aside>
<h1> – <h6> Cabeçalhos

Seis níveis de cabeçalhos de seção. <h1> é o mais importante e <h6> é o menos importante. Use em ordem hierárquica.

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Seção de nível 3</h3>
<h4>Seção de nível 4</h4>
<h5>Seção de nível 5</h5>
<h6>Seção de nível 6</h6>
Renderização

Título principal (h1)

Subtítulo (h2)

Nível 3 (h3)

<hgroup> Grupo de cabeçalhos

Agrupa um título e subtítulos relacionados como uma única unidade semântica. Útil quando um cabeçalho possui um subtítulo ou slogan.

<hgroup>
  <h1>Elementos HTML 5.0</h1>
  <p>Guia de referência completo</p>
</hgroup>
<header> Cabeçalho da seção

Representa conteúdo introdutório ou de navegação para sua seção pai. Pode conter um grupo de navegação, logotipo, formulário de busca etc.

<header>
  <img src="logo.svg" alt="Logotipo">
  <nav>...</nav>
</header>
<footer> Rodapé da seção

Representa o rodapé do conteúdo de seção mais próximo. Normalmente contém informações sobre o autor, direitos autorais ou links relacionados.

<footer>
  <p>&copy; 2025 Minha Empresa. Todos os direitos reservados.</p>
  <address>contato@empresa.com</address>
</footer>
<address> Informações de contato

Indica que o conteúdo fornece informações de contato da pessoa ou organização mais próxima — não necessariamente um endereço postal.

<address>
  <a href="mailto:contato@exemplo.com">contato@exemplo.com</a><br>
  <a href="tel:+5519999999999">(19) 99999-9999</a>
</address>
<main> Conteúdo principal

Representa o conteúdo dominante do <body>. Deve ser único por documento e não deve ser incluído dentro de <article>, <aside>, <footer>, <header> ou <nav>.

<body>
  <header>...</header>
  <main>
    <h1>Conteúdo principal</h1>
    <p>...</p>
  </main>
  <footer>...</footer>
</body>

Conteúdo de bloco (fluxo de texto)

Elementos que organizam blocos de conteúdo: parágrafos, listas, citações e divisões.

<p> Parágrafo

Representa um parágrafo de texto. Navegadores adicionam espaçamento vertical automaticamente.

<p>Este é um parágrafo de exemplo com texto simples.</p>
<p>Este é outro parágrafo separado do anterior.</p>
<hr> Separador temático

Representa uma quebra temática entre parágrafos — transição de cena, mudança de tema. Renderizado como linha horizontal.

<p>Fim de uma seção temática.</p>
<hr>
<p>Início de nova seção temática.</p>
Renderização
<pre> Texto pré-formatado

Representa texto pré-formatado. O texto é exibido com fonte monoespaçada e preserva espaços em branco e quebras de linha.

<pre>
  function ola() {
    console.log("Olá, mundo!");
  }
</pre>
<blockquote> Citação em bloco

Indica que o texto incluso é uma citação estendida de outra fonte. O atributo cite pode indicar a URL da fonte.

<blockquote cite="https://www.w3.org/">
  <p>A Web deve funcionar para todos.</p>
  <footer>— <cite>Tim Berners-Lee</cite></footer>
</blockquote>
Renderização
A Web deve funcionar para todos.
<ol> / <ul> Listas ordenada e não-ordenada

<ol> representa uma lista ordenada (numerada); <ul> representa uma lista não-ordenada (com marcadores). Ambas contêm elementos <li>.

<!-- Lista ordenada -->
<ol type="1" start="1">
  <li>Primeiro item</li>
  <li>Segundo item</li>
</ol>

<!-- Lista não-ordenada -->
<ul>
  <li>Item A</li>
  <li>Item B</li>
</ul>
<li> Item de lista

Representa um item em uma lista. Deve ser filho de <ol>, <ul> ou <menu>. O atributo value define o ordinal em listas <ol>.

<ol>
  <li value="5">Quinto item (mesmo que seja o primeiro no HTML)</li>
  <li>Sexto item</li>
</ol>
<dl> / <dt> / <dd> Lista de descrição

<dl> é a lista de descrição. <dt> é o termo e <dd> é a descrição/definição.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — linguagem de marcação padrão para a Web.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — folhas de estilo em cascata.</dd>
</dl>
Renderização
HTML
HyperText Markup Language.
<figure> Figura autocontida

Representa conteúdo autocontido — ilustração, diagrama, foto, trecho de código — opcionalmente com uma legenda via <figcaption>.

<figure>
  <img src="diagrama.png" alt="Diagrama de fluxo">
  <figcaption>Figura 1 — Diagrama de fluxo do processo.</figcaption>
</figure>
<figcaption> Legenda de figura

Representa uma legenda ou descrição para o conteúdo de seu elemento pai <figure>.

<figure>
  <pre><code>console.log("olá");</code></pre>
  <figcaption>Exemplo de console.log em JavaScript.</figcaption>
</figure>
<div> Divisão genérica

Contêiner de fluxo genérico sem significado semântico. Use apenas quando nenhum outro elemento semântico for apropriado. Amplamente usado com CSS e JavaScript.

<div class="card" id="card-principal">
  <div class="card-header">Título</div>
  <div class="card-body">Conteúdo</div>
</div>

Semântica de texto inline

Definem o significado, estrutura ou estilo de palavras, linhas ou outros fragmentos de texto.

<a> Âncora / Hyperlink

Cria um hyperlink para páginas, arquivos, endereços de e-mail, locais na mesma página ou qualquer URL. O atributo href especifica o destino.

<a href="https://www.w3.org/" target="_blank" rel="noopener">W3C</a>
<a href="mailto:contato@exemplo.com">Enviar e-mail</a>
<a href="tel:+5519999999999">Ligar</a>
<a href="#raiz">Ir ao topo</a>
<a href="arquivo.pdf" download="relatorio">Baixar PDF</a>
<em> / <strong> Ênfase e importância

<em> indica ênfase (geralmente itálico). <strong> indica importância, urgência ou seriedade (geralmente negrito).

<p>Você <em>deve</em> ler este documento.</p>
<p><strong>Atenção:</strong> esta ação é irreversível.</p>
Renderização

Você deve ler este documento.

Atenção: esta ação é irreversível.

<small> Texto pequeno / comentário

Representa comentários laterais e letras miúdas como avisos de direitos autorais, isenções de responsabilidade, termos legais.

<p>Preço: R$ 99,90 <small>(IVA incluído)</small></p>
<small>&copy; 2025 Empresa. Todos os direitos reservados.</small>
Renderização

Preço: R$ 99,90 (IVA incluído)

<s> / <u> Tachado e sublinhado

<s> representa texto que não é mais relevante ou correto (tachado). <u> representa texto com anotação não-textual — por exemplo, indicar erro ortográfico ou nome em chinês.

<p>De <s>R$ 199,00</s> por R$ 99,00</p>
<p>Este texto tem <u class="spelling-error">erro de ortografia</u>.</p>
Renderização

De R$ 199,00 por R$ 99,00

<cite> Citação de obra

Marca o título de uma obra criativa — livro, poema, filme, música, software, exposição etc.

<p>Citado em <cite>A Arte da Guerra</cite>, de Sun Tzu.</p>
Renderização

Citado em A Arte da Guerra, de Sun Tzu.

<q> Citação inline

Indica que o texto incluso é uma citação curta inline. Navegadores modernos envolvem o texto com aspas automaticamente.

<p>Como disse Einstein: <q cite="https://exemplo.com">A imaginação é mais importante que o conhecimento.</q></p>
Renderização

Como disse Einstein: A imaginação é mais importante que o conhecimento.

<dfn> Definição de termo

Marca o termo sendo definido dentro de uma frase de definição ou sentença. O title fornece a definição completa.

<p><dfn title="HyperText Markup Language">HTML</dfn> é a linguagem padrão para criar páginas web.</p>
Renderização

HTML é a linguagem padrão para criar páginas web.

<abbr> Abreviação

Representa uma abreviação ou acrônimo. O atributo title fornece a expansão ou descrição completa.

<p>O <abbr title="World Wide Web Consortium">W3C</abbr> define os padrões da web.</p>
Renderização

O W3C define os padrões da web.

<ruby> / <rt> / <rp> Anotação Ruby (pronúncia)

<ruby> representa anotações de pronúncia de caracteres do Leste Asiático. <rt> é a pronúncia; <rp> fornece parênteses para navegadores sem suporte.

<ruby>漢<rp>(</rp><rt>かん</rt><rp>)</rp>字<rp>(</rp><rt>じ</rt><rp>)</rp></ruby>
Renderização
漢(かん)字(じ)
<data> Dado com valor legível por máquina

Vincula um conteúdo a sua representação legível por máquina via atributo value.

<ul>
  <li><data value="21053">Produto A</data></li>
  <li><data value="21054">Produto B</data></li>
</ul>
<time> Data e hora

Representa um período de tempo específico. O atributo datetime deve ser um formato legível por máquina.

<p>Publicado em <time datetime="2025-04-20T09:00">20 de abril de 2025 às 9h</time>.</p>
<p>A reunião dura <time datetime="PT2H30M">2 horas e 30 minutos</time>.</p>
<code> Código-fonte

Exibe seu conteúdo em fonte monoespaçada, indicando que o texto é código de computador.

<p>Use a função <code>getElementById()</code> para selecionar um elemento.</p>
Renderização

Use a função getElementById() para selecionar um elemento.

<var> Variável matemática/programação

Representa o nome de uma variável em expressão matemática ou contexto de programação.

<p>A equação é <var>a</var><sup>2</sup> + <var>b</var><sup>2</sup> = <var>c</var><sup>2</sup>.</p>
Renderização

a2 + b2 = c2

<samp> Saída de programa

Representa saída de exemplo de um programa de computador.

<p>O terminal exibiu: <samp>Erro: arquivo não encontrado.</samp></p>
Renderização

O terminal exibiu: Erro: arquivo não encontrado.

<kbd> Entrada de teclado

Representa entrada do usuário — tipicamente texto de teclado, mas também pode ser voz ou outro dispositivo.

<p>Pressione <kbd>Ctrl</kbd> + <kbd>C</kbd> para copiar.</p>
Renderização

Pressione Ctrl + C para copiar.

<sub> / <sup> Subscrito e sobrescrito

<sub> é subscrito (abaixo da linha de base); <sup> é sobrescrito (acima).

<p>H<sub>2</sub>O é a fórmula química da água.</p>
<p>10<sup>6</sup> = um milhão.</p>
Renderização

H2O  |  106 = 1 000 000

<i> / <b> Itálico e negrito semânticos

<i> representa texto em voz ou humor alternativo — termos técnicos, estrangeirismos, pensamentos internos. <b> chama atenção sem importância extra — palavra-chave, nome de produto.

<p>O conceito de <i lang="la">tabula rasa</i> foi popularizado por Locke.</p>
<p>Adquira o <b>Produto X Pro</b> com desconto.</p>
<mark> Destaque / marcação

Representa texto marcado ou destacado por razão de referência — como resultado de busca.

<p>Resultado da busca: <mark>HTML</mark> é a linguagem da web.</p>
Renderização

Resultado: HTML é a linguagem da web.

<bdi> / <bdo> Isolamento e direção bidirecional

<bdi> isola um trecho que pode ter direção de texto diferente do seu entorno. <bdo> substitui a direção de texto atual.

<p>Usuário <bdi>مرحبا</bdi>: 10 pontos.</p>
<p><bdo dir="rtl">Este texto vai da direita para a esquerda</bdo></p>
Renderização

Este texto vai da direita para a esquerda

<span> Contêiner inline genérico

Contêiner inline genérico sem significado semântico. Útil para agrupar elementos para fins de estilo ou scripting.

<p>Minha cor favorita é <span style="color:royalblue">azul royal</span>.</p>
Renderização

Minha cor favorita é azul royal.

<br> / <wbr> Quebra de linha e oportunidade de quebra

<br> produz uma quebra de linha. <wbr> representa uma oportunidade de quebra de linha — o browser pode (mas não é obrigado a) quebrar ali.

<address>
  Rua das Flores, 123<br>
  Campinas — SP, 13000-000
</address>

<p>https://exemplo.com/um<wbr>/caminho<wbr>/muito<wbr>/longo</p>

Imagem e multimídia

Elementos para incorporar imagens, áudio e vídeo nas páginas.

<img> Imagem

Incorpora uma imagem. O atributo alt é obrigatório para acessibilidade. loading="lazy" ativa carregamento preguiçoso.

<img src="foto.jpg"
     alt="Descrição da imagem"
     width="800"
     height="600"
     loading="lazy"
     decoding="async">

<!-- Imagem responsiva -->
<img src="pequeno.jpg"
     srcset="medio.jpg 800w, grande.jpg 1600w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="Imagem responsiva">
<audio> Áudio

Incorpora conteúdo de áudio com controles nativos do navegador. Suporta múltiplos formatos via <source>.

<audio controls autoplay loop muted>
  <source src="audio.ogg" type="audio/ogg">
  <source src="audio.mp3" type="audio/mpeg">
  <p>Seu navegador não suporta áudio HTML5.</p>
</audio>
<video> Vídeo

Incorpora conteúdo de vídeo. poster exibe uma imagem antes da reprodução. playsinline evita tela cheia em iOS.

<video controls width="720" poster="capa.jpg" preload="metadata">
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="legendas-pt.vtt" srclang="pt" label="Português">
  <p>Seu navegador não suporta vídeo HTML5.</p>
</video>
<track> Faixas de texto para mídia

Especifica faixas de texto temporizadas para elementos <video> e <audio> — legendas, descrições, capítulos.

<video src="video.mp4" controls>
  <track kind="subtitles"    src="sub-pt.vtt" srclang="pt" label="Português" default>
  <track kind="subtitles"    src="sub-en.vtt" srclang="en" label="English">
  <track kind="chapters"     src="chapters.vtt" srclang="pt">
  <track kind="descriptions" src="desc.vtt" srclang="pt">
</video>
<map> / <area> Mapa de imagem

<map> define um mapa de imagem clicável. <area> define as regiões clicáveis com formas (rect, circle, poly).

<img src="mapa.png" alt="Mapa" usemap="#mapas">
<map name="mapas">
  <area shape="rect"   coords="0,0,100,100"   href="/norte" alt="Norte">
  <area shape="circle" coords="200,200,50"     href="/centro" alt="Centro">
  <area shape="poly"   coords="300,0,400,50,350,100" href="/nordeste" alt="Nordeste">
</map>

Conteúdo incorporado

Importam outro recurso ou inserem conteúdo de outra linguagem de marcação no documento.

<iframe> Frame embutido

Incorpora outra página HTML dentro da atual. sandbox restringe capacidades. allow especifica permissões.

<iframe
  src="https://www.youtube.com/embed/dQw4w9WgXcQ"
  width="560" height="315"
  title="Vídeo do YouTube"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media"
  allowfullscreen
  loading="lazy">
</iframe>
<embed> Ponto de incorporação externo

Incorpora conteúdo externo de uma aplicação ou plugin. Menos usado hoje; prefira <video> e <audio> para mídia.

<embed type="video/webm" src="clipe.webm" width="480" height="360">
<embed type="application/pdf" src="doc.pdf" width="800" height="600">
<object> Objeto externo

Representa um recurso externo que pode ser tratado como imagem, contexto de navegação aninhado ou recurso tratado por plugin.

<object data="grafico.svg" type="image/svg+xml" width="300" height="200">
  <img src="grafico-fallback.png" alt="Gráfico de barras">
</object>
<picture> Imagem responsiva

Contém zero ou mais elementos <source> e um <img> para oferecer versões alternativas de imagem para diferentes condições de exibição.

<picture>
  <source media="(min-width: 1200px)" srcset="imagem-xl.webp" type="image/webp">
  <source media="(min-width: 600px)"  srcset="imagem-md.webp" type="image/webp">
  <source srcset="imagem-sm.webp" type="image/webp">
  <img src="imagem-sm.jpg" alt="Paisagem" loading="lazy">
</picture>
<source> Fonte de mídia alternativa

Especifica múltiplos recursos de mídia para <picture>, <audio> e <video>.

<video controls>
  <source src="video.av1.mp4" type="video/mp4; codecs=av01">
  <source src="video.webm"    type="video/webm">
  <source src="video.mp4"     type="video/mp4">
</video>
<portal> Portal de navegação (experimental)

Permite incorporar outra página HTML com transições de navegação suaves. Experimental — suporte limitado.

<portal id="meu-portal" src="https://exemplo.com/pagina"></portal>
<script>
  document.getElementById('meu-portal').activate();
</script>

SVG e MathML

Incorporam gráficos vetoriais e expressões matemáticas diretamente no HTML.

<svg> Gráfico vetorial escalável

Contém gráficos vetoriais SVG inline. Permite uso de CSS, JavaScript e eventos diretamente nos elementos.

<svg viewBox="0 0 200 100" xmlns="http://www.w3.org/2000/svg" width="200" height="100">
  <rect x="10" y="10" width="80" height="80" fill="#0055aa" rx="8"/>
  <circle cx="150" cy="50" r="40" fill="#0077cc" opacity="0.8"/>
  <text x="50" y="108" font-size="12" fill="#333" text-anchor="middle">SVG</text>
</svg>
Renderização
<math> Expressão matemática (MathML)

Raiz de um fragmento MathML. Permite exibir notações matemáticas complexas nativamente no HTML5.

<math xmlns="http://www.w3.org/1998/Math/MathML">
  <mfrac>
    <mrow>
      <mo>-</mo><mi>b</mi>
      <mo>&pm;</mo>
      <msqrt>

Scripts

Definem scripts executáveis, fallbacks, templates e contextos de renderização dinâmica.

<script> Script

Incorpora ou referencia código executável — geralmente JavaScript. Atributos defer e async controlam o carregamento.

<!-- Externo com defer (executa após o parse do HTML) -->
<script src="app.js" defer type="module"></script>

<!-- Externo com async (executa assim que carrega) -->
<script src="analytics.js" async></script>

<!-- Inline -->
<script>
  document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM pronto!');
  });
</script>
<noscript> Fallback sem JavaScript

Define conteúdo alternativo exibido quando o JavaScript está desativado ou não suportado.

<noscript>
  <p><strong>Atenção:</strong> Este site requer JavaScript para funcionar corretamente.
  <a href="/sem-js">Versão sem JavaScript</a></p>
</noscript>
<template> Template de conteúdo

Mecanismo para declarar fragmentos de HTML que não são renderizados quando a página carrega, mas podem ser instanciados via JavaScript.

<template id="cartao-template">
  <div class="cartao">
    <h3 class="nome"></h3>
    <p class="descricao"></p>
  </div>
</template>

<script>
  const tmpl = document.getElementById('cartao-template');
  const clone = tmpl.content.cloneNode(true);
  clone.querySelector('.nome').textContent = 'Produto X';
  document.body.appendChild(clone);
</script>
<slot> Slot de Web Component

Define um ponto de inserção em um Web Component (Shadow DOM) onde o conteúdo fornecido pelo usuário será inserido.

<!-- No Shadow DOM do Web Component -->
<template id="meu-componente">
  <style>:host { display: block; }</style>
  <div class="wrapper">
    <slot name="titulo">Título padrão</slot>
    <slot>Conteúdo padrão</slot>
  </div>
</template>

<!-- Uso do componente -->
<meu-elemento>
  <span slot="titulo">Meu Título</span>
  <p>Meu conteúdo personalizado.</p>
</meu-elemento>
<canvas> Área de desenho bitmap

Elemento de bitmap que pode ser usado para renderizar gráficos, animações, jogos e composição de imagens via JavaScript (API Canvas 2D ou WebGL).

<canvas id="meuCanvas" width="400" height="200">
  Seu navegador não suporta canvas.
</canvas>
